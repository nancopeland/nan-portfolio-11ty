New York Magazine Perks Program

NYC-based perks program for subscribers

I recently worked on the perks program for New York magazine subscribers which offers NYC-based perks for subscribers. The perks program is supposed to be an additional benefit for current subscribers and ideally attact new subscribers.

perks redemption user experience in the New York Magazine app

Initial Ideas

The original idea for perks was that there would be two types of perks: promo code and wallet pass and the marketing team would choose which type fit best for each vendor.

The marketing team started looking into vendors and most of the initial vendors who agreed to participate were better fits for the wallet pass perk type. So, I decided to focus only on that experience for the MVP.

Then, the wallet pass vendor we were originally looking at was a bit too difficult for both product and marketing to use. It also was too expensive so we decided to design a more native redemption experience. This gave me more control over the design but also was a bit difficult because we no longer had a bar code for the vendor to scan. How can we make sure users don't redeem the perk twice?
